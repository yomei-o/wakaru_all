# わかるUNIXの歴史 ── 構想メモ（ドラフト）

「わかる」シリーズのパロディ版（＝理科ではなく計算機史）。姉妹編と同じ作法：
日常語 → 事実 → 種明かし → 正直な線の脚注 → 動く図 → 練習問題。各話HTML・印刷対応。

## 発端の問い
「Linuxカーネルは Linux 上の gcc/clang でコンパイルする。じゃあ**最初の** Linux は
何がコンパイルしたのか？」── この鶏と卵を、C言語とUNIXの歴史をさかのぼって
初代UNIXまで、そして“最初の1バイト”まで降りていく。

## 背骨（このシリーズが一貫して言うこと）
**すべてのソフトウェアは、自分より前のソフトウェアが作った。**
時間をさかのぼると、その連鎖は必ず「人間が手で彫った最初の種」と
「配線されたハードウェア」に着地する。種が撒かれた後は、
**ソフトがソフトを作るループ（自己ホスティング）**が回り続けているだけ。

合言葉：**鶏が先か卵が先か ── 最初の卵は、手で彫った。**
`bootstrap` は「自分の靴ひもで自分を持ち上げる」物理的に不可能なことの名。
最初の一回だけ、外（人間の手）が持ち上げた。

## 最重要の“種明かし”：2本の木を混同しない（第1回の核）
発端の問いで初学者が必ず混ぜる2つの直交した問い：

- **① コードの血統（遺伝子＝実際のソースコードの継承）**
  Multics（思想）→ AT&T Unix(1969) →
    ├ BSD(Berkeley) → 386BSD → NetBSD/FreeBSD/OpenBSD(1993)  ← 本物のAT&Tコードが流れる枝
    └ System V → 商用Unix
  ※ MINIX も Linux も GNU も**この血統ではない**＝白紙からの再実装（Unixのコード0行）。
- **② ツールチェーンのブートストラップ（何が何をコンパイルしたか）** ← 本シリーズの主軸

初学者の誤解「linux→minix→NetBSD」を、この2軸で解体するのが第1回の気持ちよさ。
- Linux は MINIX の**子ではなく**、MINIX という産室（開発環境）で育った（血縁なし）。
- NetBSD は Linux より**後(1993)・別の枝**。親になれない。
- 逆に MINIX 3 は後年 **NetBSD のユーザーランドを取り込んだ**（矢印は逆・ずっと後）。

## 確認済みの史実（正直な線の土台）
- Linux v0.01 は MINIX 上でしかコンパイル/テストできず、Torvalds は gcc(1.40)・bash(1.08)
  を MINIX に移植。**v0.11(1991/12) が初の自己ホスティング**（Linux上でコンパイル）。
- MINIX(1987, Tanenbaum) の純正コンパイラは **ACK（Amsterdam Compiler Kit）**、
  EM 中間バイトコード経由。MINIX 3 で NetBSD userland + Clang に移行。
- 最初の Unix(1969) は Thompson が **PDP-7 のアセンブリ**で記述。最初は
  **GE-635/GECOS 上のクロスアセンブラ（GEMAP マクロ）**でアセンブルし、
  **紙テープを物理的に PDP-7 へ運搬** → やがて自己ホスティング。
- C は B（Thompson,1969）から漸進的に育ち、B は BCPL 由来（O-code 中間コード）。
- GCC は C で書かれ、既存 cc で1回ブートストラップ後、3段ブートストラップで自作。

## 目次案（本編11＋最終＋番外）
0. 序：鶏と卵 ── Linuxカーネルは、何がコンパイルしたのか（自己ホスティングのループ提示）
1. 二本の木を混同しない ── 「血統(コード継承)」と「再実装(思想の継承)」（Tanenbaum-Torvalds論争）
2. 最初のLinux(1991) ── MINIX上のgccで生まれ、v0.11で自立した
3. GCCは何がコンパイルした? ── Cで書かれたCコンパイラの3段ブートストラップ
4. MINIXとACK ── 教育用Unixと、EM中間コードの移植性コンパイラ
5. Cコンパイラの誕生 ── BからCへ、自分で自分をコンパイルできるまで育てる（Ritchie）
6. BとBCPL ── 中間コード(O-code/EM)という「移植のための逃げ道」
7. 最初のUnix(1969, PDP-7) ── アセンブリで書かれたOS
8. アセンブラは何が作った? ── GE-635/GECOSのクロスアセンブラと、紙テープの運搬
9. 最初の1バイト ── ハンドアセンブル、トグルスイッチ、ブートストラップローダ
10. 底 ── ハードウェア：命令デコーダを配線した人間が、本当の種
- 最終回：鶏と卵の答え ── 種は手で彫られ、あとはループが回る（bootstrap＝靴ひも）

## 番外編クラスタ
- 信頼のブートストラップ①「Trusting Trust」：Thompson 1984 Turing講演。ソースを消しても
  生き残るコンパイラのトロイ。種を“信用”できるかという究極の鶏と卵。
- 信頼のブートストラップ②「Bootstrappable Builds」：GNU Mes / stage0（~数百バイトの手書き種）。
  Trusting Trustへの現代の答え＝信用する種を極小化する。①と握手。
- 「クロスコンパイル」：別CPU向けに、鶏なしで卵を作る脱出路。新アーキテクチャの立ち上げ／
  Canadian Cross。第2・8回（cross-assembler）と地続き。
- 「二本の木の続き」：BSD vs AT&T 訴訟／System V／POSIX。コードの血統と法的な鶏と卵。第1回の深掘り。
- 「なぜGNUだったのか」：自由なツールチェーンを持つための再実装（Stallman）。第3回の背景。
- 「Multics → Unix」：Unixが何への反動だったか（小さく・簡潔に）。系譜の最上流。
